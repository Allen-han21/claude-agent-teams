# 리서치/기술조사 팀 프롬프트 예제

## 기본 구성: 기술 의사결정

```
SwiftUI 마이그레이션 전략을 조사할 팀을 만들어줘:
- 현황 분석: 현재 UIKit 코드베이스 분석 (파일 수, 패턴, 의존성)
- 대안 리서치: SwiftUI best practices, 점진적 마이그레이션 전략
- 반론 담당 (devil's advocate): 마이그레이션 리스크, 비용, 대안
각자 조사 결과를 공유하고, 최종 권장안을 합의해줘.
```

### 왜 리서치에 Agent Teams가 효과적인가

리서치는 Agent Teams의 **가장 안전한 시작점**입니다:
- 코드 수정 없음 → 파일 충돌 걱정 없음
- 각자 독립적으로 조사 가능
- Devil's advocate가 편향 방지

## 변형: CLI 도구 설계

```
개발자용 TODO 트래킹 CLI 도구를 설계할 팀을 만들어줘:
- UX 담당: 사용자 경험, 명령어 인터페이스 설계
- 기술 아키텍처 담당: 파일 형식, 파싱, 스토리지
- Devil's advocate: 기존 도구와의 차별화, 단점 지적
각자 탐색 후 발견을 종합해줘.
```

## 변형: 라이브러리 평가

```
상태 관리 라이브러리를 평가할 팀을 만들어줘:
- Teammate 1: Redux Toolkit 심층 분석
- Teammate 2: Zustand 심층 분석
- Teammate 3: Jotai 심층 분석
각자 장단점, 학습 곡선, 번들 크기, 커뮤니티를 조사하고,
서로의 분석을 도전한 후 최종 추천해줘.
```

## 변형: 버그 조사 (코드 수정 없는 분석)

```
이 버그를 조사할 팀을 만들어줘 (수정은 하지 마):
- 코드 경로 추적: 관련 함수 호출 체인 분석
- 히스토리 분석: git log에서 관련 변경 이력 추적
- 환경 조사: 특정 환경에서만 재현되는지 확인
조사만 하고, 수정 방안을 보고서로 제출해줘.
```

## 팁

- **처음 Agent Teams를 사용한다면 리서치부터 시작** (공식 권장)
- Devil's advocate 역할을 반드시 포함 → 편향 방지
- "수정하지 마, 분석만" 명시하면 안전하게 실험 가능
- 결과를 마크다운 파일로 저장하도록 요청하면 재사용 가능
