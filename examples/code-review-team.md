# 코드 리뷰 팀 프롬프트 예제

## 기본 구성: 3관점 리뷰

```
PR #142를 리뷰할 에이전트 팀을 만들어줘. 3명의 리뷰어:
- 보안 관점: injection, XSS, 인증 토큰 처리, 세션 관리
- 성능 관점: 쿼리 최적화, 메모리, 알고리즘 복잡도
- 테스트 관점: 커버리지 갭, 엣지 케이스, 테스트 품질
각자 리뷰하고 심각도와 함께 결과를 보고해줘.
```

### 왜 이 구성이 효과적인가

단일 리뷰어는 한 번에 한 유형의 이슈에 집중하는 경향이 있습니다. 보안, 성능, 테스트를 독립 도메인으로 분리하면 각 영역이 동시에 철저한 검토를 받습니다.

## 변형: Plan Approval 포함

```
PR #200을 리뷰할 팀을 만들어줘.
각 리뷰어는 리뷰 전에 리뷰 계획을 제출하고, 내 승인을 받아야 해.
보안 리뷰어만 approve하고, 나머지는 리뷰 범위가 좁으면 reject해줘.
```

## 변형: 특정 모듈 집중

```
src/auth/ 디렉토리의 인증 모듈을 리뷰할 팀을 만들어줘.
프롬프트: "JWT 토큰은 httpOnly 쿠키에 저장됨.
토큰 핸들링, 세션 관리, 입력 검증에 집중."

보안 리뷰어에게는 OWASP Top 10 체크리스트를 사용하도록 지시해줘.
```

## 팁

- Lead에게 결과 종합 시 **충돌하는 의견**도 보고하도록 요청
- 리뷰 범위가 겹치지 않도록 각 리뷰어에게 명확한 렌즈 제공
- delegate 모드를 활성화하면 Lead가 직접 리뷰하지 않고 조정만 함
